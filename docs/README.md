1. 사용자의 콘솔로그에서 1~31 까지의 숫자 입력받기

```
  1. 1미만 31초과의 숫자를 입력했을 경우 "[ERROR] 유효하지 않은 날짜입니다. 다시 입력해 주세요." 출력
  2. 숫자가 아닌 것을 입력했을 경우 "[ERROR] 숫자만 입력 가능합니다." 출력
  3. 입력 받은 날짜가 1이상 25이하인 경우 '크리스마스 디데이 할인' 적용
    - 1일에 1,000원 부터 시작하여 25일까지 하루에 100원씩 증량
  4. 입력 받은 날짜가 평일(일~목)인 경우 디저트메뉴 1개당 2,023원 할인
  5. 입력 받은 날짜가 주말(금,토)인 경우 메인 메뉴 1개당 2,023원 할인
  6. 입력 받은 날짜가 일요일,크리스마스 당일인 경우 특별 할인(1,000원 할인) 적용
```

2. 사용자의 콘솔로그에서 메뉴와 메뉴의 갯수 입력받기

```
  1. - <에피타이저> 양송이수프(6,000), 타파스(5,500), 시저샐러드(8,000)
  - <메인>티본스테이크(55,000), 바비큐립(54,000), 해산물파스타(35,000), 크리스마스파스타(25,000)
  - <디저트>초코케이크(15,000), 아이스크림(5,000) <음료>제로콜라(3,000), 레드와인(60,000), 샴페인(25,000)
  - 외의 다른 메뉴를 입력했을 경우 "[ERROR] 유효하지 않은 주문입니다. 다시 입력해 주세요." 출력
  2. 메뉴-갯수 형식으로 입력하지 않았을 경우 "[ERROR] 유효하지 않은 주문입니다. 다시 입력해 주세요." 출력
  3. 메뉴의 갯수를 1 미만으로 입력했을 경우 "[ERROR] 유효하지 않은 주문입니다. 다시 입력해 주세요." 출력
  4. 메뉴 갯수의 총 합이 20개 이상인 경우 "[ERROR] 주문은 최대 20개까지 가능합니다. 다시 입력해 주세요." 출력
  5. 음료만 주문 했을 경우 "[ERROR] 음료만 주문은 불가능 합니다. 다시 입력해 주세요." 출력
  6. 입력받은 메뉴의 주문 금액을 합산
  7. 입력받은 메뉴의 총 주문 금액이 1만원 이상인 경우에만 이벤트 적용
  8. 입력받은 메뉴의 총 주문 금액이 12만 원 이상인 경우 샴페인 1개 증정
```

3. 혜택 받은 총 금액을 합산

```
    1. 혜택 총 금액이 5천원 이상인 경우 별 배지 부여
    2. 혜택 총 금액이 1만원 이상인 경우 트리 배지 부여
    3. 혜택 총 금액이 2만원 이상인 경우 산타 배지 부여
```

4. 총 주문금액에 총 혜택금액을 제외하여 결제금액 계산
